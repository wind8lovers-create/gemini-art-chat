## 概要

Gemini API を使ったチャット＆画像生成 Web アプリを **新規プロジェクト** として構築します。
前回の `chat_appri` プロジェクトのデザイン（ダークテーマ、グラスモーフィズム）は参考にしつつ、ゼロから設計します。

- **Gemini AI とのチャット**（テキスト会話＋画像生成）
- **2つの画像生成モード**（テキスト→画像、画像アップロード→修正画像）
- **チャット履歴の完全保存**（セッション単位でローカルファイルに保存）
- **画像ギャラリー**（全セッションの画像を一覧表示＆バージョン履歴）
- **Git/GitHub 連携**（自動コミット ON/OFF 切替可能、プッシュ、バージョン管理）

```
data/
├── sessions/
│   ├── {session-id}/

│   │   ├── messages.json      # チャットメッセージ一覧

│   │       └── ...
│   └── ...
└── .gitkeep
```

**metadata.json の例:**
```json
{
  "id": "session-abc123",
  "title": "猫のイラスト制作",
  "createdAt": "2025-07-14T12:00:00Z",
  "updatedAt": "2025-07-14T13:30:00Z",
  "imageCount": 3
}
```

**messages.json の例:**
```json
[
  {
    "id": "msg-001",
    "sender": "user",
    "text": "可愛い猫の絵を描いて",
    "timestamp": "2025-07-14T12:00:00Z"
  },
  {
    "id": "msg-002",
    "sender": "ai",
    "text": "可愛い猫の絵を生成しました！",
    "timestamp": "2025-07-14T12:00:05Z",
    "generatedImages": [
      {
        "id": "img-001",
        "filename": "img-001.png",
        "prompt": "可愛い猫の絵を描いて",
        "version": 1,
        "parentImageId": null
      }
    ]

## パフォーマンス最適化（重要）

Gemini Web で発生していた「画像修正を重ねると処理できなくなる」問題を、以下の3つの対策で解決します。

### 対策 1: スライディングウィンドウ方式

```
❌ Gemini Web の方式（全履歴を毎回送信）:
  → [画像1] + [画像2] + ... + [画像7] + 新しい指示 → 💥 重すぎ！

✅ 自作アプリの方式（最新だけ送信）:
  → [最新の画像] + 直近3件のテキスト + 新しい指示 → 常に軽い！
```

- API には **最新の画像 + 直近N件のメッセージ** のみを送信
- 過去の画像はすでにファイルに保存済みなので API に送る必要なし
- N は設定画面で変更可能（デフォルト: 3件）

### 対策 2: 自動セッション分割

```
セッション1: [v1] → [v2] → [v3] → [v4] → [v5]
                                              │
                    自動的に保存＆新セッション開始 ↓
セッション2: [v5を引き継ぎ] → [v6] → [v7] → ... → [v10]
                                                      │
                              自動的に保存＆新セッション ↓
セッション3: [v10を引き継ぎ] → [v11] → ...
```

- **N回修正したら自動的に新セッションを作成**（デフォルト: 5回）
- 最新の画像を新セッションに引き継ぐので、修正作業は途切れない
- 過去セッションはすべて保存済み → ギャラリーでいつでも見返せる
- ON/OFF および閾値は設定画面で変更可能

### 対策 3: チャット表示の軽量化

- チャット内の画像は **サムネイル（小さな軽い画像）** で表示
- 元画像はファイルに保存済み → クリックした時だけフルサイズ読み込み
- ブラウザのメモリを圧迫しない

## 技術的な補足

### Gemini API での画像生成の仕組み

```
ユーザー: 「可愛い猫の絵を描いて」
    ↓
API リクエスト:
  model: "gemini-2.0-flash-exp"
  contents: [{ text: "可愛い猫の絵を描いて" }]
  generationConfig: {
    responseModalities: ["TEXT", "IMAGE"]  ← これが画像生成のキー
  }
    ↓
API レスポンス:
  parts: [
    { text: "可愛い猫の絵を描きました！" },
    { inlineData: { mimeType: "image/png", data: "base64..." } }
  ]
    ↓
画像をファイルに保存 → チャットに表示
```

### 画像編集の仕組み

```
ユーザー: [猫の画像をアップロード] + 「この猫を宇宙飛行士にして」
    ↓
API リクエスト:
  contents: [
    { inlineData: { mimeType: "image/png", data: "base64..." } },  ← アップロード画像
    { text: "この猫を宇宙飛行士にして" }
